## Known limitations / non-goals

- **Single-user, single-agent** — no auth on MCP server. The model is "one developer + their agent(s) per project". Multi-tenant needs separate work.
- **No `list-scopes` CLI subcommand** — list of scopes is available via `MCP::list_scopes` and DB inspection, but not exposed as a CLI command.
- **No real-time file watching** — agent must call `track_file` after each edit. (Could be added via `notify` watcher in a future version.)
- **MCP server is stdio-only** — no HTTP transport. Fine for local agent attachment, would need SSE/HTTP for remote.
